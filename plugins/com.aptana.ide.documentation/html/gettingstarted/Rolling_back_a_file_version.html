Rolling back a file version [edit]

This page describes how to roll back a file to an earlier version using the local file history.

Introduction

You can roll back the version of a file in your projects to an earlier version by replacing the current file version with another version from the local file history.

Instructions

To roll back the version of a file:

  1. In the Project View, right-click on the file whose history you want to display, or right-click the file currently open in the editor.
  2. From the pop-up context menu, select Replace With > Local History... to open a Replace From Local History pop-up window. (To roll the file back only one version, you can select Replace With > Previous from Local History and Aptana will automatically revert the file to the previous saved version.)
  3. On the top part of the Replace from Local History window, select the date/time that corresponds to the version that you want to replace the current version with. You can preview the changes by selecting the date/time, and Aptana will display the changes in the bottom part of the window.
  4. Click the Replace button to replace the current file with the selected version.
