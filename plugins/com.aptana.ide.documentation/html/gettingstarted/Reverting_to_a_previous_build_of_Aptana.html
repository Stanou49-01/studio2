Reverting to a previous build of Aptana [edit]

This page describes how to revert your installed version of Aptana to a previous build.

Introduction

If you would like to revert your current version of Aptana to a previous build, you can revert to a previous build using the Manage Configuration window.

Note: If you had updated to a "Next" or "Nightly" build but want to switch back to the last stable build, you will need to delete the policy URL that you set. (See Changing your update version type of Aptana to find the policy URL in your Preferences), and then follow the instructions below.

Instructions

To revert to a previous build of Aptana:

  1. From the Help menu, select Software Updates > Manage Configuration to open the Product Configuration window.
  2. On the Product Configuration window, click the Revert to Previous to open a Product Configuration window with a list of previously installed versions.
  3. From the list of previous builds, select the build that you would like to revert to.
  4. Click Finish.

    Aptana prompts you to restart your workbench for the changes to take effect.

  5. Click Yes to complete the process and restart your workbench.
  6. Go back to the Help menu, and select Software Updates > Manage Configuration to open the Product Configuration window again.
  7. If you are reverting from a "next" or "nightly" build to the current "Official" build and have deleted your policy URL, you can uninstall that build so that Aptana does not prompt you to update that build.
    1. On the Product Configuration window, click the Show Disabled Features button .
    2. Expand the tree structure under Aptana IDE.
    3. Right-click the build that you reverted from and select Uninstall from the context menu, and then click OK at the prompt.

      Aptana then prompts you to restart your workbench for the changes to take effect.

    4. Click Yes to finish the uninstall.
  8. If you reverted in order to switch from a nightly build, developer branch or similar, choose Check for Aptana Updates now from the Help menu to force the update manager to check for the latest code
