Known Issues for Eclipse 3.3 users [edit]

This page contains a list of known issues for Eclipse 3.3 users who use Aptana Studio as an Eclipse plug-in.

List of Known Issues that will be fixed by Migrating to an Eclipse 3.3 base

All of the issues below will be fixed when Aptana Studio migrates to an Eclipse 3.3 base:

  • The left margin area of the editor can be colorized incorrectly in Eclipse 3.3. This is because Eclipse replaced the "column" classes for this area. This issue will be fixed when Aptana Studio moves to an Eclipse 3.3 base.
  • The Firefox preview may not work or work intermittently in Eclipse 3.3 because Eclipse 3.3 bundles XULRunner by default. Running different XULRunner versions side by side has been known to cause issues in previous releases. This issue will be fixed when Aptana Studio moves to an Eclipse 3.3 base.
  • The colorization preferences page might have multiple scroll bars because Eclipse 3.3 provides scroll bars for preference pages but 3.2 does not. Additionally, the Preferences pages might grab too much space, causing scrollbars to appear when they are not needed.

Other known issues

  • The keyboard shortcut for file upload (Ctrl+Shift+U) is mapped to another function in Eclipse 3.3. (See Trac ticket 6365.)
  • Non-project files do not automatically re-open upon re-starting Eclipse/Aptana. (See Trac ticket 5703.)
  • FTP sites and site connections are not saved upon shutdown/restart in Eclipse 3.3. (See Trac ticket 6391.)
  • Unhandled loop exception that causes crash in Eclipse 3.3. (See Trac ticket 5751.)
